## 2. 核心体验定位

玩家以林夏视角回到 2003 年中国南方县城高中废弃旧教学楼，在竖屏单手操作中用手电探索门厅、走廊和教室。恐怖感来自留白、寂静、纵向压迫、上方黑暗和物件“不对劲”，而不是血腥、鬼脸特写或连续跳吓。章节通过三段递进谜题推进：撬砖取钥匙、座位推理、课程表密码锁，最后以一次主 Jump 演出进入黑屏结算。

### 2.1 本轮重点：解谜的铺垫与难度曲线

- 本轮在「已去除可交互物常驻高亮提示框」的前提下，重做第一章三段解谜的铺垫与难度曲线，让每个谜题都有引入、有推理过程、有解开快感，**不改核心剧情走向、不改答案、不改热点判定区域与触发逻辑**，仅增加铺垫、拆分推理步骤、补充反馈演出与教学线索。
- 解谜必须「可被场景教会」：第一次使用某套规则之前，场景必须先给足教学线索，不依赖玩家外部知识。
- 难度定位：教学级偏易，但每步都要让玩家自己想一下、动一下，而不是一步到位无推理。
- 去高亮配套：所有关键线索物已在背景图中重绘得足够清晰可辨（见第 5 节），靠美术清晰度 + 旁白语言引导玩家自主观察发现；点击命中可交互物时保留「轻微一闪」即时反馈；保留可主动触发的三级提示作为兜底；设置中保留高亮提示开关，默认关闭。
- 节奏：解谜与恐怖交错——谜题 2 之后接吊扇/黑板渗水（恐怖小高潮），谜题 3 解开后接主 Jump（章节高潮）。
- 四步闭环铁律：每个谜题都必须形成「**先教学 → 再推理 → 后解锁 → 给反馈**」完整闭环，全程无死局、线索可回看。

## 4. 屏幕与适配

- 目标方向：竖屏 portrait，单手竖握。
- 目标比例与逻辑尺寸：9:16，`1080 × 1920`。
- 安全区：适配全面屏 `env(safe-area-inset-*)`；顶部按钮、底部道具栏、弹窗关闭按钮不得贴近刘海、圆角和底部手势区。
- PC 端：游戏容器居中，保持 9:16 等比缩放，左右留暗色背景。
- 移动端：保持比例信箱适配，禁止非等比拉伸；横屏时显示旋转提示遮罩。
- 输入：所有点击、拖动、滑动交互必须同时兼容鼠标和触摸，推荐统一使用 Pointer Events。

### 4.0 本轮唯一目标：统一基准的整体等比缩放（核心改动）

> 本轮**只解决**「各组件在手机端比例失调、按钮/文本框显得过大、电脑端正常」的自适应问题，让所有画面/按钮/组件按统一比例整体等比缩放。**不改任何内容、剧情、谜题逻辑、触发条件、美术资产与热点判定规则**。

#### 4.0.1 根因诊断（为什么手机端比例会错乱）

- 现状：当前 UI 组件（顶部按钮、左右箭头、对话框、道具栏、收起/展开按钮、弹窗、密码锁、飘字等）大量使用 `vw / vh / clamp(... vw ...)` 这类**相对“浏览器视口”**的单位定义尺寸与字号。
- 问题：`vw` 取自视口宽度、`vh` 取自视口高度。手机端（窄高、19.5:9 等）与电脑端（宽屏、留白后舞台很高）下，同一段 `vw/vh` 折算出的物理像素与“9:16 舞台画面”之间的比例关系完全不同——于是出现「手机端按钮/文本框相对画面格外大」「某些元素缩了、某些没缩」的比例错乱。
- 结论：**必须让所有组件尺寸不再相对视口，而是相对统一的 9:16 基准舞台整体等比缩放。**

#### 4.0.2 统一缩放基准（设计基准画布）

- 设计基准画布固定为竖屏 **9:16 = `1080 × 1920`**。
- 所有画面、按钮、文本框、道具栏、图标、字号、内边距、间距、圆角、阴影偏移等尺寸，**全部以基准画布像素为单位定义**，再乘以统一缩放系数。
- **禁止**任何游戏内组件直接使用写死的固定像素（如 `min-height: 62px`、`padding: 42px`、`font-size: 25px` 等绝对 px），也**禁止**继续用相对视口的 `vw/vh`（除舞台容器自身的尺寸计算外）。

#### 4.0.3 统一缩放系数 `--stage-scale`（机制核心）

- 设“基准舞台容器”即现有 `#game-container`，严格保持 9:16，用 `width: min(100vw, calc(100vh * 9 / 16))`、`height: min(100vh, calc(100vw * 16 / 9))` 实现 contain 适配（保持比例、不裁切、不拉伸），多出的空间由 `#game-shell` 的纯黑/暗色背景自然形成 letterbox 留边。
- 由 JS 在初始化、`resize`、`orientationchange` 时，根据容器**实际渲染宽度**计算统一缩放系数并写入 CSS 变量：
  ```js
  // 容器实际宽度 / 基准画布宽度 1080 = 统一缩放系数
  const rect = gameContainer.getBoundingClientRect();
  const scale = rect.width / 1080;            // 因容器严格 9:16，用宽或高算结果一致
  gameContainer.style.setProperty('--stage-scale', scale);
  // 同时提供一个“1 基准像素 = ?px”的换算变量，便于 calc 使用
  gameContainer.style.setProperty('--upx', (rect.width / 1080) + 'px');
  ```
- CSS 中所有组件尺寸统一改写为：`calc(基准像素值 * var(--stage-scale))`，或等价地 `calc(基准像素值 * var(--upx))`（推荐用 `--upx`，即“单位基准像素”，写法更直观：`min-height: calc(62 * var(--upx));`）。
- 结果：电脑端 / 手机端 / 各种手机比例下，**所有组件相对基准画布的比例完全固定**，只有整体随屏幕大小等比放大或缩小，**组件之间的相对大小、间距、位置在任何设备上完全一致**。

#### 4.0.4 字体同步缩放

- 所有文字字号同样基于基准画布定义并乘以同一系数：`font-size: calc(基准字号 * var(--upx));`。
- 取消所有 `clamp(... vw ...)` 形式的字号，改为单一的“基准字号 × `--upx`”，确保字号与组件、与画面整体严格同步缩放，不再单独漂移。
- 字号基准值沿用文档既有数值（如对话框正文 `25` 基准像素、飘字字号见第 11 节飘字表），只是把单位从 px 改成 `× var(--upx)`。

#### 4.0.5 安全区与留边处理

- 舞台等比缩放后多出的空间一律用纯黑/暗色 letterbox 留边（由 `#game-shell` 背景承担），不得拉伸舞台填满。
- 所有 UI 与可交互内容**必须落在 9:16 基准舞台内**，不切边、不进刘海/灵动岛/底部 home 条；全面屏安全区通过对舞台容器整体施加 `env(safe-area-inset-*)` 内缩处理，**不破坏组件之间的等比关系**（即安全区只影响舞台外边距，不单独缩放某个组件）。
- 边距、间距铁律按“基准画布像素”衡量：按钮距画面边缘 ≥ `20` 基准像素、按钮间距 ≥ `10` 基准像素，缩放后由系数统一带动，物理边距随屏幕等比变化但视觉占比恒定。

#### 4.0.6 热点与点击同步缩放（保证“画面所见 = 可点所在”）

- 现有热点/Canvas 输入已通过 `container.getBoundingClientRect()` 把物理坐标按 `rect.width/height` 映射回 `1080×1920` 逻辑坐标（见 `js/input.js` 的 `toGamePoint`），**这套映射本身是等比的**——只要舞台严格保持 9:16，缩放后点击位置依然精准，本轮**不改热点坐标数据与判定逻辑**。
- 要求：本轮调整布局/缩放后，必须复测热点点击在电脑端与各手机比例下仍精准命中，确保缩放不引入偏移。若舞台容器尺寸计算有任何改动，需同步保证 `toGamePoint` 的 `rect` 取自同一个严格 9:16 容器。

- **禁止对立绘容器使用固定宽高同时赋值**（如 `width:1080px; height:1920px`）而不保持比例——必须以一维为基准、另一维度自适应。
- 推荐实现：Canvas 绘制立绘时，计算 `srcW/srcH` 与目标区域的比例，取 `min(targetW/srcW, targetH/srcH)` 为缩放系数，居中绘制。
- 突脸放大演出：放大动画仅改变 `scale` 系数（整体放大），不单独修改宽或高，保证等比。
- 各手机比例（18:9/19.5:9/20:9/4:3）下立绘比例必须正常、关键部位（头部/上半身）不被裁切。

## 6. 章节流程

### 6.1 开始与教学

1. 加载页显示“正在校准手电……”和进度。
2. 开始界面显示标题《空教室》、开始、读档、声音开关。
3. 点击开始后解锁音频，循环播放雨声环境音和低频 BGM。
4. 首次进入门厅显示匿名照片叙事，引导玩家拖动手电、切换左右视角、点击热点。
5. 前 30 秒必须让玩家获得正向反馈：拾取半截钢筋、查看匿名座位表照片或看到热点显影。

### 6.1.1 开场前置剧情段落（本轮新增 · 轻动画式）

> **插入位置**：仅在「点击开始 → 进入门厅」之间插入此段，**不改动门厅及之后任何逻辑**。**读档进入时跳过前置剧情**，直接恢复到存档状态。
>
> **设计目标**：氛围从平静缓缓滑向不安，做氛围与动机铺垫，**不剧透『林夏锁门』真相**。**总时长控制在 40~60 秒，全程可随时跳过**。

#### 触发与流程衔接

1. 玩家在主菜单点击「开始」并解锁音频后，**先进入开场剧情段落**，而非直接进入门厅。
2. 开场剧情结束（或玩家点击「跳过」）后，**画面缓缓淡入门厅场景 `scene_01_lobby_gate`，与现有 `LOBBY` 流程无缝衔接**，后续门厅匿名照片叙事、谜题 1 等逻辑完全不变。
3. 从主菜单点击「读档」进入时，**直接跳过本段**，进入存档恢复流程。
4. 新增状态标志 `flags.introPlayed`：本段播放完成或被跳过后置 `true` 并写入存档；标志为 `true` 时不再自动重播（玩家主动「重来」回到本章起点时可重新播放）。

#### 全局演出（贯穿三个画面）

- **背景音**：进入开场即低音量循环播放 [低频压迫 BGM](./assets/audio/bgm_low_pressure_loop.mp3)（音量 `0.18`）+ [雨声环境](./assets/audio/ambience_rain_thunder_creak.mp3)（音量 `0.28`）+ [磁带沙沙杂音](./assets/audio/intro_tape_hiss.mp3)（音量 `0.30`，循环）。
- **噪点/雪花**：画面边缘叠加轻微雪花噪点层（alpha `0.06~0.12` 缓动呼吸），呼应老旧磁带画质，不可过强干扰阅读。
- **跳过按钮**：右下角固定一个**老式按钮风格**的「跳过 ▷」按钮（旧塑料/做旧金属质感，非现代扁平 UI），距画面边缘 ≥ `20px`，尺寸约 `220×96`，半透明常驻，点击立即停止全部剧情动画与音频淡出、淡入门厅。同时兼容鼠标与触摸。
- **逐字旁白**：旁白文字逐字浮现（字速参考 `28 字/秒`），位于画面下方安全区，半透明黑底衬底、思源宋体，文字不贴边、不超屏；**每段旁白完整显示后至少停留 1s** 再随画面切换。
- **轻动画手法约束**：只用「图片缓动（平移/缩放/淡入淡出）+ 噪点/雨丝叠加 + 文字逐字浮现」实现，**不要求复杂逐帧**，避免卡顿与生硬硬切；画面之间用 `≥600ms` 交叉淡入过渡。

#### 三个画面节拍

**画面1 · 回忆/现实交界（约 13~18s）**

- 背景：`intro_01_tape.png`（黑场中浮现旧磁带特写 + 林夏成年女性的手放入随身听）。
- 入场动画：黑场 → 磁带特写以 `1.06→1.00` 缩放 + alpha `0→1` 缓慢浮现（约 `1.2s`）。

- 轻动画：旁白末尾播放 [磁带按下播放键咔哒](./assets/audio/intro_tape_play_click.mp3)，随后磁带齿轮区域做缓慢循环旋转的轻微位移/明暗呼吸（模拟齿轮转动），画面边缘雪花噪点轻微增强。

**画面2 · 声音引入（约 13~18s）**

- 背景：`intro_02_walkman.png`（随身听播放、雨夜虚化）。
- 过渡：画面1 交叉淡入画面2（`≥600ms`）。
- 音效：随身听传出沙沙杂音（intro_tape_hiss 提至 `0.40` 短暂强调后回落），叠加 [极轻女声广播残响『……今天的午间广播……』](./assets/audio/intro_voice_broadcast_faint.mp3)（极轻、失真、被杂音半掩）。
- 旁白（逐字）：『可它昨天，自己寄回到了我手上。』
- 轻动画：随身听指示灯做暗哑锈红 `#9E2B2B` 微弱明灭（周期约 `1.2s`，alpha `0.3↔0.85`）；背景雨夜虚化层做极缓慢横向平移（视差 `4~8px`），营造朦胧不安。

**画面3 · 动机交代（约 14~22s）**

- 背景：`intro_03_school_rain.png`（雨夜废弃教学楼剪影 + 倒计时牌 + 一扇窗微光）。
- 过渡：画面2 交叉淡入画面3（`≥600ms`）。
- 镜头：背景以 `1.00→1.05` 极缓慢推近（Ken Burns 式缓动，约 `6~8s`），镜头朝教学楼剪影推进。
- 轻动画：前景叠加斜向飘落的雨丝层（半透明白线，持续向下平移循环）；教学楼那扇窗口的冷青光 `#C8D4D0` 做极缓慢明灭（呼应主题，暗示『不该亮的光』，**但不点破真相**）。
- 旁白（逐字）：『清和中学……苏晚……我得回去一趟。哪怕只是为了，再听一次那天到底发生了什么。』
- 结束衔接：旁白停留 ≥ `1s` 后，画面**缓缓淡入门厅场景 `scene_01_lobby_gate`**，磁带杂音/广播残响淡出，雨声与 BGM 平滑过渡到门厅，无缝进入正式游戏。

#### 时长与节奏控制

- 三画面合计目标 **40~60 秒**；若实现偏长，可按 1.1.1 比例压缩各画面停留，但**旁白可读性优先，关键旁白显示不少于其逐字时长 + 1s 停留**，不得为压缩时长牺牲可读性。
- 全程任意时刻点击「跳过」立即结束并淡入门厅。

- 走廊视角：`scene_03_corridor_depth`（走廊纵深·含教室门/机油/公告栏）、`scene_03b_corridor_tools_corner`（清洁工具角落·含水龙头/机油特写）、`scene_04_corridor_fan_up`（抬头吊扇·纯氛围）。

## 8. 核心系统需求

### 8.1 手电筒光柱

- 初始光心：`x=690,y=1080`，模拟右手拇指持握。
- 输入：Pointer Down/Move 拖动光心；无拖动时缓慢回到右下默认位置。
- 光照范围：内圈半径 `110`，中圈半径 `260`，外圈半径 `520`。
- 暗区：外圈外 alpha `0.76~0.86`，保持冷青灰压暗。
- 光色：中心暖黄 `rgba(232,200,122,0.50)`，边缘渐隐。
- 限制：光心 y 坐标默认限制在 `260~1480`，避免遮挡道具栏。
- 主 Jump：手电自动收窄至半径 `180`，锁定苏晚剪影。

### 8.2 热点与高亮

- 探索态默认不显示常驻高亮描边/标签（沿用 0.2.7），玩家靠观察重绘后清晰的背景线索物 + 旁白语言引导自主发现可交互点。
- 点击命中可交互物：触发一次「轻微一闪」即时反馈（短暂描边 + 暖光晕）+ 点击音效，不常驻显示。
- “显影”按钮作为兜底安全网：点一下临时闪现当前视角全部可交互热点轮廓（一次性，冷却 `3s`，非常驻）。
- 三级提示作为兜底，由玩家主动逐级触发：①旁白方向引导 →②文字规则提示 →③给出推理逻辑/计算并对相关热点短暂一闪（一次性）。逐级展开、不限次数（详见第 9 节各谜题）。
- 高亮提示开关在设置中保留，默认关闭；开启后恢复常驻高亮，照顾不同玩家。
- 成功交互：播放对应 SFX、出现粒子、飘字、自动存档。
- 错误交互：显示短句提示，不扣资源，不造成死局。

### 8.3 道具栏与线索册

- 底部条状道具栏固定 6 槽，拾取后按剧情顺序排序。
- 点击道具进入详情：放大图、名称、描述、线索文本、使用、组合、关闭。
- 使用道具：支持先选道具再点热点，也支持点热点后提示选择匹配道具。
- 组合接口：本轮保留，不设置强制组合；半条友情手链 + 磁带 A 面可触发额外回忆文本。
- 线索册：所有关键文本自动加入，任意时刻可回看，确保谜题无死局。

### 8.4 逐字对话框

- 位置：屏幕下方，不遮挡道具栏。
- 字速：普通叙述 `28 字/秒`；苏晚台词 `16 字/秒`。
- 可读性：关键提示至少停留 `1.0s`；点击可快速显示整句，再点击关闭。
- 苏晚台词：轻微抖动 `1~2px`，颜色在 `#C8D4D0` 与低透明 `#9E2B2B` 之间缓慢漂移。

### 8.5 三位密码锁

- 三位数字 `0-9` 循环滚动。
- 按钮：确认、清空、关闭。

## 9. 谜题设计与提示（四步闭环重构）

> 本章为本轮核心改动。所有谜题统一遵循「**先教学 → 再推理 → 后解锁 → 给反馈**」四步闭环。在「已去除常驻高亮」前提下，靠**旁白语言引导**玩家自主观察重绘后清晰的背景线索物，靠**分步推理**避免一步到位，靠**反馈演出**给足解开快感。
>
> 共同铁律：
> - 答案不直接写在线索旁；规则在玩家第一次使用前由场景先教会；线索与谜题同处一条逻辑链、当前场景即可获取，不依赖外部知识与记忆。
> - 旁白引导只指方向、不报答案（如“看看墙边/讲台/黑板”），把“发现热点—理解规则—得出结论”留给玩家。
> - 三级提示改为**玩家主动触发**的兜底：① 旁白方向引导 → ② 文字规则提示 → ③ 给出推理逻辑/计算方式并对相关热点短暂一闪（一次性，非常驻）。逐级展开、不限次数。
> - 点击命中任意可交互物时保留「轻微一闪」即时反馈（沿用 0.2.7 的 tapFeedback），不常驻显示。


### 9.0 锈门谜题（走廊级 · 轻量「观察探索+材料常识推理」· v1.3 新增）

> **定位**：走廊段轻量谜题，难度轻、预期耗时 2~3 分钟、无死局、有试错反馈。与谜题 1「撬砖+钥匙」明显区分，核心是「材料常识判断（油润滑/水助锈）」，而非道具组合。**进入教室的唯一且必经路径。**

- 场景：走廊纵深（scene_03）、走廊清洁工具角落（scene_03b）。
- 道具：机油⑧（用后即消耗，不滞留道具栏）。
- 唯一解：机油润滑门轴/锁孔 → 门开；水助锈为陷阱，试错不惩罚。

**① 先教学（铺垫 + 方向引导）**
- 玩家点击走廊尽头教室门：门轴锈死推不开；旁白（林夏口吻）："门轴锈死了，硬推没用……得让它松一松。"
- 旁白方向引导："走廊里应该有能用的东西……看看角落。"——引导玩家探索走廊两侧。

**② 再推理（两处可用之物 · 材料常识判断）**

走廊布置两处可交互物（热点），玩家需自主判断：

| 物件 | 位置 | 性质 | 交互结果 |
| --- | --- | --- | --- |
| 生锈滴水的水龙头（水） | scene_03b 清洁工具角落 | **陷阱** | 若玩家"用水"：旁白冷冷提示"水只会让锈咬得更死……"；门纹丝不动；不惩罚、可继续尝试 |
| 落灰的缝纫机油/机油瓶 | scene_03b 清洁工具角落 | **正解** | 拾取机油（飘字 `+机油`）；再对教室门使用 → 解锁 |
| 旧公告栏 | scene_03 走廊右侧墙面 | **氛围伏笔** | 点击弹出公告栏图 `clue_bulletin_board.png`；「午间广播·播音员」一栏名字被水渍遮住看不清；旁白：「这一栏的名字，看不清了。」；**④ 林夏内心独白（点击公告栏查看时）**：「广播站……她最喜欢的地方。那时候我要是多陪她一会儿就好了……」（随即打住、不再多言）；**不参与任何谜题，玩家不看也能通关** |

**「水=陷阱」反向设计（关键体验）：**
- 若玩家先用水：门纹丝不动，旁白："水只会让锈咬得更死……"（冷静，不嘲讽）。
- 试错不惩罚、可继续尝试，目的是让玩家自己悟出「油润滑、水助锈」的材料常识。
- 试错后可继续拾取机油使用正解，无死局。

**③ 后解锁**
- 玩家拾取机油 → 对教室门使用 → 门轴/锁孔涂油 → 门「吱呀——」松动推开。
- 机油使用后即从道具栏消耗，不滞留。
- 门开后点击教室门/门口热点正式进入教室（`transitionToGroup('classroom', ...)`）。

**④ 给反馈（解开快感）**
- 开门：播放 [生锈门锁打开](./assets/audio/sfx_rusty_lock_open.mp3) + 屏幕轻抖 `2px/2帧` + 门后教室冷气/光涌出效果（画面短暂亮度提升 `120ms`）+ 飘字「门开了」。
- 旁白一句压抑收束（如："教室里没有人。第十三排，却有一张桌子干净得像刚被擦过。"）。

**公告栏广播伏笔（氛围增量·非线索）：**
- 走廊（清洁工具角附近）放置褪色旧公告栏，贴一张广播站节目单/值班表。
- 「午间广播·播音员」一栏名字被水渍/划痕盖住、看不清，只埋悬念不点破。
- 与已锁定的「广播刺啦响女声半句」「苏晚=广播员」呼应，增强信息密度与恐怖氛围。
- **明确：此公告栏不参与任何谜题，玩家不看也能正常通关。**

**三级提示（玩家主动触发）：**
1. 旁白引导："走廊角落里也许有能用的东西……润滑的东西。"
2. 文字规则："门轴生锈，需要润滑剂。水只会让锈更严重——找找有没有油。"
3. 推理逻辑 + 一闪：对 scene_03b 的**机油瓶**短暂一闪，并给出——"清洁工具角落有瓶机油，拾取后对教室门使用，门轴润滑后就能推开。"

### 9.1 谜题 1：撬砖取钥匙（教学级 · 教会“自主观察场景→发现热点→道具组合”）

- 场景：门厅正面铁门（scene_01）、门厅侧墙（scene_02）。
- 道具：半截钢筋、生锈钥匙。唯一解：无钢筋不可撬砖；无钥匙不可开门；钢筋使用后不丢弃。

**① 先教学（引入 + 方向引导）**
- 玩家先点门厅铁门锁，发现门锁死、打不开。旁白触发引入：“门锈死了，硬拉没用。**得找个能撬动它的东西**……先看看左右两边的墙。”
- 该句引导玩家用左右箭头/滑动切换到门厅侧墙，建立“环视场景找工具”的基本操作认知。

**② 再推理（自主观察 + 组合）**
- 切到侧墙后，背景图中**墙边半截钢筋（明显冷色金属反光）**与**下方松动砖（明显凸出、砖缝深阴影、在整面墙中唯一）**均清晰可辨，无需高亮即可被发现。
- 玩家先拾取钢筋（飘字 `+半截钢筋`）。
- 点松动砖（未持钢筋时）：旁白“这块砖松了，**但手指抠不动，得用根硬东西撬**。”——提示玩家“砖+工具”的组合关系，但不替玩家操作。
- 玩家自行用钢筋对松动砖使用，完成“观察→热点→道具组合”。

**③ 后解锁**
- 撬砖成功：砖被撬开，砖后露出生锈钥匙；玩家拾取钥匙后回到铁门，用钥匙开锁，进入走廊。

**④ 给反馈（解开快感）**
- 撬砖：播放 [钢筋撬砖](./assets/audio/sfx_rebar_pry_brick.mp3) “咔啦”声 + 砖缝灰尘飘落粒子（见 11 节，新增“撬砖灰尘”）+ 屏幕轻抖 `2px/2帧` + 钥匙特写小弹窗（放大旋转入袋）+ 飘字 `+生锈钥匙`。
- 开锁：播放 [生锈门锁打开](./assets/audio/sfx_rusty_lock_open.mp3) + 闪白 `120ms` + 旁白“……开了。”

**三级提示（玩家主动触发）：**
1. 旁白引导：“试试看墙边有没有能用的东西。”
2. 文字规则：“墙角那块砖好像松动了，光靠手撬不开，需要一件硬工具。”
3. 推理逻辑 + 一闪：对侧墙的**钢筋**与**松动砖**短暂各闪一次，并给出组合方式——“先拾取半截钢筋，再对松动砖使用它，砖后藏着钥匙。”

### 9.2 谜题 2：座位推理（逻辑级 · 教会“交叉多条线索定位”）

- 场景：教室入门全景（scene_05）、双桌特写（scene_06）、黑板特写（scene_07）、讲台门后（scene_08）。
- 必需线索（三条，缺一不可交叉）：点名册、值日表、烧角座位表照片。

**① 先教学（引入 + 依次方向引导）**
- 进教室后，旁白依次引导玩家注意三处线索源（每条引导只指方向、不报内容），引导玩家自行走查点击：
  - “讲台上那本**点名册**还摊着……过去看看名字。”（点名册在 scene_07 讲台桌面已重绘清晰可读）
  - “门后钉着一张**值日表**，纸都发黄了。”（值日表清晰可读）
  - “手里这张**烧角的座位表照片**，再看一眼。”（道具栏照片可放大查看）

**② 再推理（分三步交叉，玩家自己拼）**
- 第①步——点名册：读出 `13号 苏晚`，得到“苏晚 = 13 号”。
- 第②步——值日表：读出规则“靠窗第一列从前排开始编号，**单号坐前排**，单号沿第一列向后递增”，得到“定位方法”。
- 第③步——综合：13 是单号，沿靠窗第一列从前排向后数到第十三排——推出“苏晚 = 靠窗那一列、单号、靠前排的那张干净课桌”，并与烧角照片上“被锈红圈住、旁残留‘夏’”的第十三排双桌对应。
- 推理由玩家交叉三条线索自己得出，文档不在任一线索旁直接写出座位答案。

**③ 后解锁（强制顺序不变）**
- 当玩家综合三条线索后点对干净课桌：镜头推向那张干净课桌，林夏旁白“……是这里。我怎么会忘。”
- 强制顺序：必须**先打开林夏积灰旧抽屉**（scene_06 左桌，背景已重绘可见“林夏”刻字），获得半条友情手链 + 磁带 A 面；**再面对苏晚干净课桌**（进入谜题 3）。
- 顺序未到时点苏晚课桌：旁白“那张桌子太干净了，**像在等你先想起自己的位置**。”——引导先去开林夏抽屉，不卡死。

**④ 给反馈**
- 每读到一条新线索：[热点发现与点击](./assets/audio/sfx_hotspot_click.mp3) + 线索入册飘字 + “线索 1/3 → 3/3”进度低语提示（仅显示已集齐进度，不剧透答案）。
- 三条集齐 + 定位正确：解谜成功粒子（见 11 节）+ 镜头推近干净课桌 + 上述林夏旁白（停留 ≥1s）。
- 开林夏抽屉：[旧抽屉吱呀](./assets/audio/sfx_drawer_creak.mp3) + 飘字 `+半条友情手链`/`+磁带 A 面`。

**恐怖交错（谜题 2 之后）：** 完成座位推理后，触发走廊吊扇异常（scene_04）与黑板渗水（scene_07）的恐怖小高潮，再进入谜题 3，保证“解谜与恐怖节奏交错”。

**三级提示（玩家主动触发）：**
1. 旁白引导：“再仔细看看讲台、门后和手里的照片。”
2. 文字规则：“把**学号、列数、单双号、前后顺序**综合起来看，别只盯着一条。”
3. 推理逻辑图 + 一闪：依次对点名册、值日表、座位表照片各闪一次，并给出逻辑链——“苏晚是 13 号（单号）→ 靠窗第一列、单号坐前排向后数 → 第十三排靠窗那张干净课桌；先开积灰的林夏抽屉，再面对苏晚课桌。”

### 9.3 谜题 3：课程表密码锁（进阶级 · 先教规则再让玩家算）

## 10. 关键演出数值

### 10.1 走廊吊扇广播

1. 切到 `scene_04_corridor_fan_up` 后停顿 `0.8s`。
2. 播放吊扇嘎嘎声，吊扇区域叠加轻微旋转阴影，速度 `6deg/s`。
3. `1.4s` 后播放广播刺啦。
4. `2.0s` 后播放“今天的午间广播……”，随后戛然而止。
5. 屏幕轻抖 `2px/3帧`，解锁教室。

### 10.2 黑板渗水字

- 首次进入或再次聚焦 `scene_07_blackboard_close` 触发渗水演出。
- 文字：“连你也松开手了，是不是？”必须直接绘制在黑板背景图中，不使用系统文字、对话框、纯文本叠字或浮空字。
- 美术表现：歪扭白色粉笔字，带粉笔颗粒、断笔、手写歪斜、湿痕遮罩并融入黑板。
- 动态表现：粉笔字缝隙中出现水痕，`6.2s` 内缓慢下流；结束后保留低强度湿痕循环，支持反复观看。
- 音频表现：触发时播放极轻水滴音效，并保留全局低频 BGM 压迫氛围。

### 10.3 主 Jump（本轮更新：新增突脸演出段落）

> 本轮在原有流程基础上，在剪影浮现后新增「突脸爆发」段落（使用 `suwan_jumpscare_closeup_transparent.png`），与常态安静形成强反差，时长极短（0.3~0.8s），点到为止。

1. `0ms`：暂停 UI，禁用点击，播放关窗砰声。
2. `0~180ms`：黑幕 alpha 从 `0` 到 `1`，画面执行 `5帧×2组`短抖，最大 `8px`。
3. `300ms`：广播刺啦低音量进入，BGM 降至 `0.05`。
4. `650ms`：黑幕中画面压暗至近黑。
5. **`900~2100ms`：苏晚浮现段（`suwan_emerge_pose_transparent.png`）**：从 `y=860,alpha=0` 浮现到 `y=480,alpha=0.92`，等比 contain 缩放，不得横向压扁。
6. `950ms`：播放"你……终于……回来了……夏夏"（声音由委屈转失真尖锐）。
7. `1200~2500ms`：执行 `5帧×4组`快速抖动，每组最大 `8px`；移动端震动 `[120,80,180,60,220]`。
8. **`2550ms`：突脸爆发段（`suwan_jumpscare_closeup_transparent.png`）**：
   - 浮现图立即淡出，突脸图从 `scale=0.6,alpha=0` 极速推进至 `scale=1.05,alpha=1.0`，持续 `300~500ms`。
   - 全程等比缩放，不得横向压扁。
   - 同时：画面执行 `5帧×3组`剧烈抖动，单组最大 `10px`；广播刺啦音量拉至峰值；移动端再次震动 `[150,60,200]`。
9. `3050ms`：突脸图与全部画面同时急速黑屏（alpha `1→0`，约 `80ms`）。
10. `3400ms`：进入黑屏结算。

**突脸节制原则：**
- 突脸总时长 `0.3~0.8s`（步骤8），作为爆发性高潮，不重复触发，不延长。
- 突脸演出结束后立即黑屏，不停留在狰狞特写上。
- 本轮只在主 Jump 这一处使用突脸，不新增其他惊吓点。

9. `3400ms`：进入黑屏结算。
